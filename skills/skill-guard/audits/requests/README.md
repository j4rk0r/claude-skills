# Audit Request Queue

## How to request an audit

1. Fork this repo
2. Create a JSON file here named `{skill-name}.json`:

```json
{
  "skill": "author/repo@skill-name",
  "source": "github.com/author/repo",
  "requested_by": "your-github-username",
  "date": "YYYY-MM-DD",
  "reason": "Installing for project X, want security review first"
}
```

3. Open a PR with the title: `audit-request: {skill-name}`

## What happens next

- A maintainer runs `skill-guard` on the requested skill
- The audit is generated by the system (never from user-submitted results)
- The result is published to `audits/{author}/{skill-name}/`
- Your request file is removed and the PR is closed

## Rules

- **Only request files are accepted** — PRs that add or modify audit result JSONs will be rejected
- **One skill per request** — submit separate PRs for multiple skills
- Requests are processed in order of submission
